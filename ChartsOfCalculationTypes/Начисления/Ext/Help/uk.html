План видов расчета «Основные начисления организации»

Содержит список начислений, используемых при учете заработной платы работников организаций. В этот план видов расчета включаются все начисления.

В системе есть предопределенные элементы, которые не могут быть удалены из списка начислений.

Наряду с предопределенными элементами в план видов расчета можно добавлять свои виды расчетов, указывая у каждого порядок его налогообложения налогом на доходы физ. лиц, а также способ отражения начисленных сумм в бухгалтерском и налоговом учете. Для этого предназначена специализированная форма диалога.

Форма диалога разделена на 4 страницы: Основная, Формула, Параметры базы, Дополнительно.

Страница "Основная"

Реквизиты этой страницы разбиты на следующие группы:

Параметры учета для расчета НДФЛ

- в этой группе один реквизит "НДФЛ". Тут надо выбрать вид дохода к которому будет отнесено начисление, от значения этого реквизита зависит порядок расчета НДФЛ, а также то каким образом доход будет отражен в отчете Форма 1-ДФ.

Параметры учета для расчета индексации

- в этой группе два реквизита "Социальная группа" и "Индексируется расчетом". Эти реквизиты могут быть и не заполнены, если текущий вид расчета не является базой ни для одной из индексаций. Но если  реквизит "Социальная группа" заполнен, то надо прописать текущий вид расчета в качестве базового для вида индексации, который указан в реквизите "Индексируется расчетом" (это можно сделать на странице "Параметры базы" в таблице "Является базой для").

Параметры для расчета премий

- в этой группе необходимо указать, является ли вид расчета премией и при необходимости - число месяцев, за которые производится начисление премии.

Специализированные параметры начислений

Флаг "Это отпускные" - если установлен, то текущий вид расчета будет доступен в форме выбора из документа начисления отпуска.

Показатели видов оплаты труда

Реквизиты этой группы определяют порядок отражения начислений по этому виду расчета в в статистических отчетах.

Учет отработанного времени

Эти реквизиты заполняются в случае если вид расчета принимает участие в учете времени. Список для реквизита "Обозначение в табеле учета рабочего времени:" может быть откорректирован пользователем в соответствии с принятыми на предприятии буквенными обозначениями учета времени.

Важно! Если надо указать невыход, который уменьшает число календарных дней в году для отпуска, то, хотя этот вид расчета и влияет на учет времени, не надо устанавливать ни один из флагов. Но надо указать этот вид расчета для начислений, которые он вытесняет в таблице "Вытесняющие расчеты" на странице "Параметры базы".

Страница "Формула"

Здесь необходимо задать последовательность расчета и способ расчета.

Есть ряд начислений, которые при расчете в пределах одного расчетного периода должны использовать результаты расчетов других начислений этого же расчетного периода. Для осуществления этой процедуры все начисления разбиваются на группы: первичные, зависимые первого уровня, зависимые второго уровня и зависимые третьего уровня. Затем виды расчетов работника группируются и расчет происходит в 4 этапа, рассчитываем по порядку каждую группу.

Способ расчета определяет схему по которой будет произведен расчет. Можно выделить две группы схем: предопределенные способы расчетов и произвольная формула (см. Редактор формул расчета).

Предопределенные виды расчетов:

Оклад по дням - рассчитывается на основании месячной тарифной ставки, невыходы учитываются по дням. Если сотрудник отработал 15 дней из 21 рабочего дня в месяце, то сумма начисления будет рассчитана по формуле ("Месячная ставка")/21 *15;

Оклад по часам - рассчитывается на основании месячной тарифной ставки, невыходы учитываются по часам. Если сотрудник отработал 120 дней из 167 рабочих часов в месяце, то сумма начисления будет рассчитана по формуле ("Месячная ставка")/167 *120;

Оплата по дневному тарифу - расчет на основании дневного тарифа ("Дневная ставка" * 15)

Оплата по часовому тарифу - расчет на основании часового тарифа ("Часовая ставка" * 120)

Доплата за ночные часы, Доплата за вечерние часы - для расчета этих начислений необходимо, чтобы в графике расчетов присутствовали ночные и вечерние часы. При заполнении начислений в реквизиты расчета будет заполнена часовая ставка текущего месяца и процент (ночных/ вечерних). Часовая ставка для сотрудников у которых основным начислением являются "Оклад по дням " или "Оклад по часам" часовая ставка расчитывается по формуле "Месячный тариф" / "Норма часов за месяц". Для "Оплата по дневному тарифу" расчет по формуле "Дневной тариф" / 8.

Индексация зарплаты, Индексация начислений за счет фонда социального страхования, Индексация начислений за счет фонда социального страхования от несчастного случая - алгоритм расчета индексации можно представить в виде формулы Сумма базы * Коэффициент индексации.

  1. Сумма базы - так как индексируются доходы населения в пределах прожиточного минимума, установленного для соответствующих социальных и демографических групп населения  в конфигурации предусмотрено перечисление Социальные группы населения.
  2. Расчет коэффициента индексации - коэффициент рассчитывается на основании двух показателей:
    • публикуемого каждый месяц индекса потребительских цен (хранится в регистре сведений Индекс инфляции) .Индексы инфляции должны вносится пользователем самостоятельно ежемесячно;
    • рассчитываемого прироста этого индекса. Для расчета прироста используется так называемая Дата обнуления базового индекса (хранится в регистре сведений Коэффициенты повышения окладов, изменяется кадровыми документами)  

Замещение и Совмещение - результат расчета нулевой, это начисление используется для осуществления правильного учета времени в случае когда работник не выполнял свои основные обязанности а замещал кого либо. В этом случае среди начислений появятся начисления по замене у которых в поле сотрудник будет заполнен сотрудник который замещал, а в поле "Назначение" сотрудник которого замещали. Записи появятся по основному начислению сотрудника, которого замещали а так же по всем его штатным надбавкам

Прогул - результат расчета нулевой, используется для организации правильного учета времени

Оплата по среднему БЛ, Оплата по среднему БЛ (за счет организации), Оплата по среднему БЛ (травма на производстве), Оплата по среднему БЛ (по беременности и родам) - можно рассчитывать только в специализированном документе Начисление по больничному листу

Отпуск очередной, Отпуск за свой счет, Компенсация отпуска, Возврат отпускных при увольнении - можно рассчитывать только в специализированном документе Начисление отпуска работникам организаций

Доплата до среднего заработка и Командировка - можно рассчитывать только в специализированном документе Оплата по среднему заработку. Для вида расчета в этом документе происходит только расчет средней, в дальнейшем в документе Начисление зарплаты работникам организаций должна появится соответствующая этому начислению запись, в которой и будет произведен окончательный расчет. 

Премия (Месячная), Премия (Квартальная), Премия (Годовая) - это премии которые начисляются как процент от базовых начислений. Так как базовым периодом для этих видов расчета считаются прошлые периоды (месяц, квартал и год соответственно), то для этих начислений установлена категория "Первичное". Если на предприятии существует премия, для котой необходимо использовать начисления текущего месяца, то для этого надо самостоятельно создать вид расчета указав вид премии "Является премией (за текущий месяц)" либо "Является премией (за текущий месяц), которая учитывается пропорционально" и определить для него категорию "зависимые первого уровня" (или выше)

Отпуск по уходу за ребенком - результат расчета нулевой, служит для учета времени

Оплата по договору подряда - это начисление будет заполнятся только при наличии документа Договор на выполнение работ с физ. лицом. В документе Начисление зарплаты работникам организаций обязательно заполняется пара Вид расчета и Документ основание. При расчете анализируются параметры Документа основания и производится расчет.

 

Страница "Параметры базы"

На странице четыре таблицы:

  • Базовые расчеты - заполняется только в случае, когда для формулы необходимо использовать базу;
  • Ведущие расчеты - эта таблица используется для механизма перерасчетов;
  • Вытесняющие расчеты - здесь указываются расчеты, которые вытесняют текущий;
  • Являются базой для - в этой  таблице надо перечислить все взносы, в базу которых входит текущее начисление; другие начисления (например индексация); а также определить входит ли текущее начисление в базу средних.

    • Флаг База по назначению - если установлен, то при получении базы будет анализироваться суммы по соответствию паре реквизитов Сотрудник и Назначение (например внутреннему совместителю назначена премия процентом от оклада за текущий месяц только по должности на которой он работает как внутренний совместитель).Если флаг не установлен, то для базы будут анализироваться все начисления (примером такого начисления может служить индексация).

    Страница "Дополнительно"

    На странице обязательно указывается Счет учета, а в случае кода всем работникам надо формировать одинаковые проводки по текущему начислению надо заполнить реквизиты Способ отражения в бухучете и Способ отражения в упр.учете.
