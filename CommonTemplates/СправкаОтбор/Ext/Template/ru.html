Отбор в стандартных отчетах

Для чего нужен отбор?

На закладке отбор задаются условия для ограничения данных, выводимых в отчет.

Например, при указанной настройке отбора в отчет попадут данные только по номенклатуре "Телевизор "JVC".

Какие условия ограничивают выводимые данные?

В отчет выводятся данные, удовлетворяющие каждому условию помеченному флажком.

Например, при указанной настройке отбора в отчет попадут данные по номенклатуре "Телевизор "JVC" и организации "Белая акация".
Строка склад Равно "Главный склад" не учитывается, т.к. она не помечена флажком.

На какие поля можно можно ставить условия в отборе?

Условия можно ставить на предложенные по умолчанию поля, их реквизиты и другие доступные поля для отбора, которые можно выбрать в форме "Выбор поля".

Например, при указанной настройке отбора в отчет попадут данные с номенклатурой, которая не является услугой.

 

Что такое Тип сравнения?

В колонке "Тип сравнения" указывается тип сравнения поля со значением, укзанным в колонке "Значение".

"Тип сравнения" может быть:

  1. Равно - значение поля должно совпадать со значением условия
  2. Не равно - значение поля должно не совпадать со значением условия
  3. Меньше - значение поля должно быть меньше значения условия
  4. Меньше или равно - значение поля должно быть меньше или равно значению условия
  5. Больше - значение поля должно быть больше значения условия
  6. Больше или равно - значение поля должно быть больше или равно значения условия
  7. Интервал (>, <) - значение поля должно быть в интервале двух значений условия
  8. Интервал (>=, <=) - значение поля должно быть в интервале включая границы двух значений условия
  9. Интервал (>=, <) - значение поля должно быть в интервале включая левую границу двух значений условия
  10. Интервал (>, <=) - значение поля должно быть в интервале включая правую границу двух значений условия
  11. Содержит - значение поля должно содержать в себе строку, указанную в значении условия
  12. В списке - значение поля должно присутствовать в значении условия
  13. В иерархии - значение поля должно либо совпадать, либо присутствовать в папке, указанной в значении условия
  14. В списке по иерархии (см. пример) - значение поля должно совпадать с любым значением из списка или распологаться в любой папке, указанной в значении условия.
  15. Не в списке - значение поля должно отстутствовать в списке значений условия
  16. Не в списке по иерархии - значение поля должно отсутствовать в списке и папках, указанных в значении условия
  17. Не в иерархии - значение поля должно должно не совпадать и отсутствовать в папке, указанной в значении условия 
  18. Не содержит - значение поля не должно содержать в себе строку, указанную в значении условия

Например, при указанной настройке отбора в отчет попадут данные со всей номенклатурой , которая находится в папках "Товары" и "Услуги".
